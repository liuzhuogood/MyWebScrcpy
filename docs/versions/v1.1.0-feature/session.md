# v1.1.0-feature 会话记录

## 2026-07-11

### 用户需求

1. **新增大屏展示页面**：创建一个大屏/仪表盘页面，展示所有已连接手机的屏幕画面（类似监控墙），并在大屏上支持对每台设备进行触控操作。
2. **头部 Logo 优化**：在项目头部左上角添加一个 Logo 图标。

### 需求分析

- **大屏页面**：需要一个新页面（如 `/dashboard.html`），通过多个 WebSocket 连接同时接收所有设备的视频流，以网格/瀑布流布局展示每台设备的屏幕画面。每台设备的画面需要独立解码（复用现有 `ScrcpyDecoder`），并支持基础的触控操作（点击、滑动）。
- **Logo**：在 `index.html` 的 `.header` 区域添加一个 Logo 图标（可以是 SVG 或图片），位于左上角。

### 设计决策

- 大屏页面独立于 `index.html` 和 `player.html`，作为第三个页面
- 使用 CSS Grid 布局，根据设备数量自动调整列数
- 每个设备卡片包含：画面 canvas、设备名称、连接状态
- 触控操作复用 `ControlPacker` 的触摸事件打包逻辑
- Logo 使用 SVG 内联方式，避免外部依赖
