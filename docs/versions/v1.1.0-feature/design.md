# v1.1.0-feature 设计文档

## 背景

MyWebScrcpy 当前提供两个页面：
- `index.html`：设备列表页，展示所有已连接设备的卡片信息，点击进入单设备播放器
- `player.html`：单设备播放器页面，支持完整的触控操作、全屏、工具栏等

用户需要一个"大屏展示页面"，可以同时监控所有已连接设备的屏幕画面，类似于监控中心的多画面墙。同时，用户希望在头部添加 Logo 提升品牌识别度。

## 目标

1. **大屏展示页面（Dashboard）**
   - 展示所有已连接设备的实时屏幕画面
   - 网格/瀑布流布局，根据设备数量自适应
   - 每个设备画面独立解码和渲染
   - 支持基础触控操作（点击、滑动）
   - 显示设备名称和连接状态
   - 自动刷新设备列表（设备上下线时更新）

2. **Logo 优化**
   - 在 `index.html` 头部左上角添加 Logo 图标
   - 保持与现有 UI 风格一致

## 非目标

- 不改变现有 `player.html` 的功能
- 不修改 Go 后端逻辑（前端直接通过 WebSocket 连接获取视频流）
- 不实现大屏上的完整工具栏（如返回键、Home 键等）

## 关键决策

### 1. 页面架构

新增 `/dashboard.html` 页面，与 `index.html`、`player.html` 并列。

### 2. 视频流获取

复用现有 WebSocket 协议：
- 每个设备建立独立的 WebSocket 连接：`/ws?serial={serial}`
- 接收 meta 消息获取视频参数（codec、width、height）
- 接收二进制帧进行解码
- 通过 WebSocket 发送触控控制消息

### 3. 布局方案

使用 CSS Grid 实现自适应网格：
- 1 台设备：1 列
- 2 台设备：2 列
- 3-4 台设备：2 列
- 5-6 台设备：3 列
- 7+ 台设备：3 列

每个设备卡片包含：
- Canvas 画面（保持原始宽高比）
- 设备名称（底部半透明遮罩）
- 连接状态指示

### 4. 触控操作

复用 `ControlPacker` 类，为每个设备创建独立的 packer 实例：
- 鼠标点击 → 触摸 DOWN/UP
- 鼠标移动（按下时）→ 触摸 MOVE
- 触摸事件（移动端）
- 坐标映射：根据 canvas 显示尺寸和设备实际尺寸计算

### 5. Logo 方案

使用内联 SVG 图标，避免外部依赖：
- 位置：`.header` 内最左侧
- 样式：与现有 UI 风格一致（蓝色调）
- 尺寸：24x24px 或 32x32px

## 风险与权衡

### 风险
1. **性能**：多个设备同时解码视频流可能消耗大量 CPU/GPU 资源
   - 缓解：限制最大显示设备数量（如 9 台），超出时提示用户

2. **网络带宽**：多个视频流同时传输
   - 缓解：大屏模式下使用较低分辨率（可通过 scrcpy 参数控制，但当前版本不修改后端）

3. **WebSocket 连接数**：每个设备一个连接
   - 缓解：浏览器通常支持 6 个并发连接（HTTP/1.1），WebSocket 没有此限制

### 权衡
- 牺牲部分触控功能（无工具栏按钮）换取更简洁的大屏体验
- 使用 CSS Grid 而非更复杂的拖拽调整布局，降低实现复杂度

## 待确认问题

1. Logo 具体使用什么图标？（默认使用 📱 + 项目名的 SVG 图标）
2. 大屏页面是否需要从设备列表页跳转？（默认在设备列表页添加"大屏模式"按钮）
