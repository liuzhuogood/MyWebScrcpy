# v1.1.0-feature 版本说明

## 版本信息

- **版本号**：v1.1.0
- **发布日期**：2026-07-11
- **版本类型**：功能版本（Feature）

## 变更内容

### 新增功能

1. **大屏展示页面（Dashboard）**
   - 新增 `/dashboard.html` 页面，可同时展示所有已连接设备的屏幕画面
   - 手机外观仿真：每台设备以仿真手机形态展示（圆角边框、听筒、Home 条）
   - Flex-wrap 自适应布局：手机窗口自动排列，填满一行后换行
   - 小/中/大三档尺寸切换（localStorage 记忆上次选择）
   - 每个设备画面独立解码，实时显示视频流
   - 支持基础触控操作（点击、滑动、滚轮、右键返回）
   - 每个设备自带工具栏：返回主页、切换任务、电源键、全屏
   - 自动检测横屏设备并旋转手机框（横竖屏混合排列不重叠）
   - 自动刷新设备列表（设备上下线时自动更新，5 秒轮询）
   - 断线自动重连（最多 10 次）

2. **设备列表页入口**
   - 在 `index.html` 头部添加"大屏模式"按钮，点击跳转到大屏页面

### 优化改进

1. **Logo 显示**
   - 在 `index.html` 头部左上角添加项目 Logo 图标（内联 SVG）
   - 大屏页面头部同样展示 Logo

### 文档

1. **README 更新**
   - 添加大屏监控截图
   - 特性列表增加"大屏监控模式"
   - 项目结构增加 dashboard.html 和 dashboard.js

## 技术细节

- 大屏页面复用现有的 `ScrcpyDecoder` 和 `ControlPacker` 类
- 使用 Flexbox + flex-wrap 实现自适应排列（放弃 Grid 布局方案）
- 每个设备建立独立的 WebSocket 连接获取视频流
- ResizeObserver 监听容器尺寸变化，自动重算 canvas
- 触控操作通过 WebSocket 发送 scrcpy 控制协议消息

## 兼容性

- 不影响现有 `index.html` 和 `player.html` 的功能
- 不修改 Go 后端逻辑
- 需要浏览器支持 WebCodecs API（Chrome 94+）
