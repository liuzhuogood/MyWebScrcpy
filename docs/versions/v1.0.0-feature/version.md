# 版本说明

**发布日期**：2026-07-10

### 版本概述

MyWebScrcpy v1.0.0 是首个正式发布版本，提供了完整的基于浏览器的 Android 屏幕镜像和远程控制功能。该版本实现了零客户端安装的核心目标，用户只需打开浏览器即可镜像和控制任何 ADB 连接的 Android 设备。

### 主要功能

1. **实时屏幕镜像**
   - 基于 WebCodecs API 的硬件加速视频解码
   - 支持 H.264、H.265、AV1 多种编解码器
   - 可配置视频参数（分辨率、码率、帧率）
   - 自动适应设备屏幕旋转和分辨率变化

2. **设备管理**
   - 自动 ADB 设备发现（每 2 秒轮询）
   - 设备信息显示（型号、序列号、产品名）
   - 在线/离线状态过滤
   - 多设备支持（独立会话管理）

3. **远程控制**
   - 鼠标控制：左键点击/拖拽、右键返回、滚轮滚动
   - 键盘控制：文本输入、特殊按键、修饰键支持
   - 触摸控制：移动浏览器触摸事件（支持压力感应）
   - 工具栏：Home、Back、Recents、Power、Rotate、Fullscreen

4. **连接可靠性**
   - 自动重连（指数退避，最多 20 次尝试）
   - 断线检测和通知
   - 服务器启动重试机制
   - 帧读取超时检测

5. **用户体验**
   - 响应式设计（移动端优化）
   - 加载状态指示
   - 错误信息显示
   - 屏幕关闭检测
   - FPS 计数器显示
   - 全屏模式支持（含 iOS 伪全屏）

### 技术实现

**后端架构**：
- Go 语言实现，单二进制部署
- WebSocket 双向通信
- scrcpy 4.0 协议完整实现
- 内嵌 scrcpy-server JAR 和前端资源

**前端架构**：
- 原生 JavaScript，无框架依赖
- WebCodecs API 硬件加速解码
- Canvas 视频渲染
- 响应式 CSS 设计

**协议支持**：
- scrcpy 4.0 协议
- 控制消息：触摸、滚动、按键、文本注入
- 视频消息：配置包、关键帧、Delta 帧
- 会话管理：旋转/分辨率变化检测

### 环境要求

**服务端**：
- Go 1.25.5 或更高版本
- ADB 已安装并配置
- Android 设备已启用 USB 调试

**客户端**：
- 现代浏览器（Chrome 94+、Firefox 93+、Safari 16.4+、Edge 94+）
- 支持 WebCodecs API 的浏览器

### 配置参数

**环境变量**：
- `PORT`：监听端口（默认 8080）
- `ANDROID_HOME`：ADB 路径（可选）

**视频参数**（默认值）：
- 最大分辨率：1024 像素
- 码率：2 Mbps
- 最大帧率：15 FPS
- 编解码器：H.264
- 控制：启用
- 音频：禁用

### 已知限制

1. **音频支持**：当前版本不支持音频传输
2. **多设备管理**：每次只支持一个设备的镜像会话
3. **安全性**：WebSocket 连接未加密，建议在受信任网络中使用
4. **性能**：默认配置优化为办公场景，不适合高性能游戏流
5. **录制功能**：不支持屏幕录制或截图保存

### 部署方式

**单二进制部署**：
```bash
# 构建
go build -o mywebscrcpy .

# 运行
./mywebscrcpy
# 或指定端口
PORT=9090 ./mywebscrcpy
```

**访问地址**：
- 设备列表：http://localhost:8080
- 播放器页面：http://localhost:8080/player.html?serial=<设备序列号>
